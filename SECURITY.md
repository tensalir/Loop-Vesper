# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in Loop-Vesper, please report it responsibly.

**Do not open a public GitHub issue for security vulnerabilities.**

Instead, please contact the maintainers directly with:

1. A description of the vulnerability
2. Steps to reproduce
3. Potential impact
4. Any suggested fixes (optional)

## Supported Versions

We provide security updates for the latest version on the `main` branch.

## Security Best Practices

When contributing, please ensure:

- No secrets or API keys are committed to the repository
- Environment variables are used for sensitive configuration
- Input validation is performed on user-supplied data
- Dependencies are kept up to date
